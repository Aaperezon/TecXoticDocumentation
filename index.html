Software: Philosophy and Abstract.

Innovation requires technological implementation and human ideals, that is why TecXotic’s software philosophy is to develop and use modular subsystems that work independently from others, thus making a scalable and simple to use system.
TecXotic’s team developed and implemented its own software from the principle of the client-server architecture. This model gives rise to data sharing, data processing, and data centralization which makes it safe for data integrity.

Software: Main technologies

There are several technologies implemented for the development of KOLOP. Starting with the operative systems (Windows 10 and Raspberry Pi OS), the main systems that carry the software development. The implementation of code in both OS and the protocols involved are essential for complete interaction with KOLOP

Unity: The User Interface (UI) for the complete control of KOLOP was developed using this tool. It allows us to create visual indicators, view cameras, and send control instructions.

C#: It is used to create the client-side connection of the client-server architecture implemented to communicate the UI and KOLOP and control the visual indicators in the UI. It allows the connection over TCP to send instructions in JSON format and to receive real-time stream video from the cameras using UDP.

Python: It is used to create the server-side connection of the client-server architecture implemented to communicate the UI and KOLOP and activate or deactivate preprogrammed instructions for controlling and using Artificial Intelligence (AI).

MAVLink: Along with the Python programming language, it is used to send manual and autonomous control instructions to the motors of KOLOP.

OpenCV: Along with the Python programming language, it is used to recognize images from the cameras and to achieve an autonomous and intelligent behavior from KOLOP to complete the tasks.

GStreamer: It is used to create a real-time stream video over UDP from the cameras as the source. It allows the developed systems to connect and use the streaming video either for viewing or for image processing.

Software: Communication flowchart

There are three main protocols for communicating the subsystems: UDP, TCP, and MAVLink.
1.- UDP is used for real-time streaming of the camera images;
2.- TCP, for data streaming and sending instructions from the UI to KOLOP;
3.- and MAVLink to send messages to the Pixhawk, thus controlling the roll, pitch and yaw for manual and autonomous control, as well as to get data from the sensors in the Pixhawk.
